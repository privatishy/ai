# 🚀 Руководство быстрого старта — PKAI

## Требования

- ✅ **Python >= 3.10** — [Скачать](https://www.python.org/downloads/)
- ✅ **Веб-камера или IP-камера** — любая камера с поддержкой UVC/OpenCV
- ✅ **4+ GB RAM, 10+ GB свободного места на диске**
- ✅ **(Опционально) GPU NVIDIA** — для ускорения инференса (RTX 3050 и выше)

> 💡 **Примечание**: Система работает на CPU (4-5 FPS). GPU не обязателен для разработки и тестирования, но рекомендуется для продакшн-развёртывания.

---

## Быстрый запуск (5 минут)

```bash
# 0. Убедитесь, что камера подключена и не используется другими программами
#    - Закройте Zoom, Skype, OBS, если они захватывают камеру
#    - Проверьте работу камеры в стандартном приложении "Камера"

# 1. Клонирование репозитория
git clone https://github.com/privatishy/ai.git
cd ai

# 2. Создание виртуального окружения
python -m venv venv

# 3. Активация окружения
# Windows (PowerShell / CMD)
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate

# 4. Установка зависимостей
pip install -r requirements.txt

# 5. Копирование конфигурации
# Windows PowerShell
Copy-Item .env.example .env
# macOS / Linux
cp .env.example .env

# 6. Запуск системы
python main.py
```

## Первые шаги в интерфейсе

### 1. Проверка видеопотока

- Откройте http://localhost:5000
- Убедитесь, что в окне отображается видео с камеры
- В углу должно отображаться значение FPS (4-5 на CPU)

### 2. Проверка распознавания

- Поместите в кадр любой объект (чашку, книгу, поднос)
- Наблюдайте за обновлением списка «Распознанные блюда»
- Проверьте, что общая стоимость пересчитывается динамически

> ⚠️ **Важно**: На данном этапе модель предобучена на датасете COCO (общие объекты). Распознавание школьных блюд начнётся после дообучения на собственном датасете.

### 3. Работа с админ-панелью

- Перейдите на http://localhost:5000/admin
- Проверьте раздел «Статистика системы» (время работы, FPS)
- Протестируйте кнопку «📥 Экспорт логов» — скачается ZIP-архив с логами

### 4. Проверка API

- Откройте в браузере: http://localhost:5000/api/detect
- Убедитесь, что возвращается JSON-ответ:

```json
{
  "dishes": [],
  "total": 0,
  "fps": 4.2,
  "objects": 0
}
```

## Переменные окружения

Файл `.env` уже создан с безопасными настройками по умолчанию:

```env
# Модель
MODEL_PATH=weights/yolov8m.pt
MODEL_CONFIDENCE=0.5
MODEL_IMAGE_SIZE=640

# Камера
CAMERA_ID=0
CAMERA_WIDTH=1920
CAMERA_HEIGHT=1080

# Сервер
FLASK_HOST=0.0.0.0
FLASK_PORT=5000
FLASK_DEBUG=False

# GPU
USE_CUDA=False
CUDA_DEVICE=0
```

Вам не нужно ничего менять для локальной разработки!

### Когда редактировать .env:

| Сценарий                          | Что изменить                                  |
| --------------------------------- | --------------------------------------------- |
| Используется вторая камера        | CAMERA_ID=1                                   |
| Требуется более низкое разрешение | CAMERA_WIDTH=1280, CAMERA_HEIGHT=720          |
| Запуск на удалённом сервере       | FLASK_HOST=0.0.0.0 (уже установлено)          |
| Включить отладку Flask            | FLASK_DEBUG=True                              |
| Использовать GPU NVIDIA           | USE_CUDA=True (требуется CUDA-версия PyTorch) |

## Диагностика проблем

### 🎥 Камера не определяется

**Симптомы:**

- Ошибка: «Не удалось открыть камеру»
- Чёрный экран в интерфейсе

**Решение:**

1. Проверьте, что камера не используется другими программами (Zoom, Skype, OBS)
2. Убедитесь, что CAMERA_ID в `.env` соответствует нужному устройству:

```bash
# Проверка доступных камер (Python)
python -c "import cv2; [print(f'Camera {i}: {cv2.VideoCapture(i).isOpened()}') for i in range(4)]"

```

3. На Windows: попробуйте установить CAMERA_BACKEND=cv2.CAP_DSHOW в `config.yaml`

### 🐌 Низкая производительность (FPS < 5)

**Это нормально для CPU-режима.**

| Режим      | Ожидаемый FPS | Оборудование                |
| ---------- | ------------- | --------------------------- |
| CPU        | 4-5           | Любой современный процессор |
| GPU (CUDA) | 30+           | NVIDIA RTX 3050 и выше      |

**Для ускорения:**

1. Установите CUDA-версию PyTorch:

```bash
pip install torch==2.0.1 torchvision==0.15.2 --index-url https://download.pytorch.org/whl/cu118
```

2. В `.env` установите: USE_CUDA=True
3. Перезапустите систему

### 🤖 Модель не загружается

**Симптомы:**

- Ошибка: FileNotFoundError: weights/yolov8m.pt

**Решение:**

1. Убедитесь, что файл модели находится в папке `weights/`
2. При первом запуске модель скачивается автоматически — проверьте подключение к интернету
3. Проверьте путь в MODEL_PATH файла `.env`

### 📁 Логи не записываются

**Решение:**

1. Убедитесь, что папка `logs/` существует:

```bash
mkdir logs
```

2. Проверьте права на запись в папку
3. Убедитесь, что в `.env` указан корректный путь: LOG_FILE=logs/app.log

### 🔌 Порт 5000 занят

**Симптомы:**

- Ошибка: OSError: [Errno 98] Address already in use

**Решение:**

1. Найдите процесс, использующий порт:

```bash
# Windows
netstat -aon | findstr :5000
# macOS / Linux
lsof -i :5000
```

2. Или измените порт в `.env`:

```env
FLASK_PORT=5001
```

3. Доступ к приложению: http://localhost:5001

## Режим разработки

Для удобства отладки можно включить режим разработки Flask:

1. В `.env` установите:

```env
FLASK_DEBUG=True
```

2. Запустите с автоперезагрузкой:

```bash
python main.py
```

**Преимущества режима отладки:**

- Автоматическая перезагрузка при изменении кода
- Детальные сообщения об ошибках в браузере
- Интерактивный отладчик

> ⚠️ **Не используйте FLASK_DEBUG=True в production!**
