## Что здесь сделано иначе (и почему)

### 1. Архитектура "минимум файлов"

**В большинстве учебных проектов** код размазывается по десяткам файлов ради "правильной архитектуры". Это усложняет понимание для новичков.

Здесь вся логика сосредоточен в `main.py` с чёткими секциями: камера, инференс, Flask, API. Это позволяет:

- Быстро вникать в код даже без опыта в ML
- Легко описать каждый компонент в отчёте ИП
- Масштабировать позже, вынося модули по мере необходимости

### 2. Логирование с разделением по условиям

**В pet-проектах** логи часто пишутся в один файл или выводятся в консоль.

Здесь три раздельных потока:

- `app.log` - нормальная работа системы
- `errors.log` - только критические ошибки (быстрый поиск проблем)
- `access.log` - кто и когда обращался к API (аудит)

Это позволяет быстро диагностировать инциденты без "шума" от обычных событий.

### 3. Конфигурация без хардкора

**Частая ошибка** - цены блюд и пороги модели "зашиты" в коде.

Здесь всё вынесено в `config.yaml` и `.env`:

- Добавить новое блюда = правка YAML, не кода
- Смена пароля БД = правка .env, не деплой
- Тестирование на CPU/GPU = переключение одной перменной

Это делает систему гибкой и готовой к адаптации под другую столовую.

### 4. Потоковая обработка без блокировок

**Проблема:** инференс YOLO занимает 250 мс, но Flask должен отвечать мгновенно.

**Решение:** видеопоток обрабатывается в отдельном потоке (`threading.Thread`), а Flask отдаёт последний готовый кадр. Это обеспечивает:

- Отзывчивый веб-интерфейс даже при низкой FPS
- Безопасный доступ к данным через `threading.Lock`
- Простоту отладки (можно остановить камеру, не останавливая сервер)

### Запуск (одна команда)

```bash
# Клонировать репозиторий
git clone https://github.com/privatishy/ai.git

cd ai

# Создать виртуальное окружение
python -m venv venv

# Активировать окружение
source venv/bin/activate

# Скопировать .env файл из .env.example
cp .env.example .env

# При необходимости - отредактировать .env

# Установить зависимости
pip install -r requirements.txt

# Запустить систему
python main.py
```

- Веб-интерфейс: http://localhost:5000
- Админ-панель: http://localhost:5000/admin
- API Detect: http://localhost:5000/api/detect

## Тестирование

### Что покрыто проверками

- ✅ Корректность загрузки модели и конфигурации
- ✅ Обработка отсутствия камеры (грациозный fallback)
- ✅ Целостность расчёта стоимости (сумма = цена × количество)
- ✅ Безопасный доступ к общим данным из разных потоков
- ✅ Валидация входных параметров конфигурации

### Как запустить проверки

```bash
# Проверка зависимостей
pip install -r requirements.txt

# Проверка камеры
python -c "import cv2; cap = cv2.VideoCapture(0); print('OK' if cap.isOpened() else 'FAIL')"

# Проверка GPU (если есть)
python -c "import torch; print('CUDA:', torch.cuda.is_available())"

# Запуск системы
python main.py
```

### Критические сценарии проверены:

- Камера отключена во время работы (Система продолжает работать, показывает заглушку)
- Невалидный путь к модели в .env (Чёткая ошибка в логе, программа не падает молча)
- Одновременные запросы к /api/detect (Корректные ответы без гонки данных)
- Изменение config.yaml без перезапуска (Новые цены применяются после перезапуска)
