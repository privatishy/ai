> Это backend-система **автоматизации оплаты питания для школьной столовой**,

> делать в реальном продакшене: с логированием, безопасностью и масштабируемостью.
> Проект про аккуратную архитектуру, устойчивость к ошибкам и понятный код.

## О проекте

PKAI - работающая система автоматизации процесса оплаты питания в школьной столовой. Проект использует нейросетевую модель YOLOv8m для распознавания блюд на подносе в реальном времени с точностью >92%.

**Автор:** Кулик Арсений Олегович, 10 класс "Е"
**Научный руководитель:** Деев Андрей Павлович
**Учебное заведение:** ЧОУ "Газпром школа Санкт-Петербург"
**Год проекта:** 2025-2026

Что я делал осознанно:

- ✅ **анализировал** требования реальной школьной столовой,
- ✅ явно **фиксировал** все допущения в конфигурации,
- ✅ **проектировал** систему так, чтобы её можно было масштабировать.

### Ключевые возможности
- Распознавание блюд: Real-time детекция через веб-камеру (YOLOv8m).
- Авторасчёт: Мгновенный подсчёт стоимости заказа на основе цен.
- Веб-интерфейс: Live-видео поток (MJPEG) + статистика в браузере.
- Админ-панель: Управление моделью, экспорт логов, системные события.
- Логирование: Разделные логи (app, errors, access) для аудита.
- REST API: Эндпоинты для интеграции с внешними системами.
- Дизайн: Стиль ЧОУ "Газпром школа Санкт-Петербург"

## Технический стек

### Backend & AI
- **Python 3.10+** (strict type hinting ready)
- **YOLOv8m** (Ultralytics) - баланс скорости и точности
- **PyTorch 2.0.1** - движок нейросети
- **OpenCV 4.8.1** - обработка видеопотока
- **Flask 3.0.0** - легковесный веб-сервер

### Frontend
- **HTML5/CSS3/JS** (Vanilla) - без тяжелых фреймворков
- **MJPEG Stream** - низколатентная передача видео

### Инфраструктура и инструменты
- **YAML** - декларативная конфигурация классов
- **python-dotenv** - управление секретами (.env)
- **Logging** - многоуровневая система логирования
- **Threading** - асинхронная обработка камеры и HTTP

### Запуск (одна команда)
```bash
# Клонировать репозиторий
git clone https://github.com/privatishy/ai.git

cd ai

# Создать виртуальное окружение
python -m venv venv

# Активировать окружение
source venv/bin/activate

# Скопировать .env файл из .env.example
cp .env.example .env

# При необходимости - отредактировать .env

# Установить зависимости
pip install -r requirements.txt

# Запустить систему
python main.py
```

### После запуска:
- Веб-интерфейс: http://localhost:5000
- Админ-панель: http://localhost:5000/admin
- API Detect: http://localhost:5000/api/detect

## Использование

**Главная страница (`http://localhost:5000`):**
- Видеопоток с камеры в реальном времени (MJPEG)
- Список распознанных блюд с количеством и ценами
- Общая стоимость заказа в реальном времени
- Техническая статистика (FPS, количество объектов)

**Админ-панель (`http://localhost:5000/admin`):**
- Статистика системы и нагрузки
- Управление моделью (дообучение, кэш)
- Эскпорт логов для аудита
- Системные события и ошибки

---

## API Endpoints
- `GET /` - Главная страница
- `GET /admin` - Админ-панель
- `GET /video_feed` - MJPEG поток видео
- `GET /api/detect` -  JSON с данными распознавания
- `GET /api/export-logs` - Экспорт логов (ZIP) 
- `POST /api/retrain` - Запуск дообучения
- `POST /clear-cache` - Очистка кэша

**Привер ответа API (`/api/detect`):**
```bash
{
   "dishes": [
      {"name": "borsch". "count": 1, "price": 85},
      {"name": "pasta", "count": 1, "price": 65}
   ],
   "total": 150,
   "fps": 4.2,
   "objects": 2
}
```

## Конфигурация
**Переменные окружения (.env):**
```bash
MODEL_PATH=weights/yolov8m.pt
MODEL_CONFIDENCE=0.5
CAMERA_ID=0
FLASK_PORT=5000
USE_CUDA=False
```

**Классы распознавания (config.yaml):**
Система поддерживает гибкую настройку классов и цен без изменения кода.
1. `borsch` (борщ) — 85 руб.
2. `pasta` (макароны) — 65 руб.
3. `cutlet` (котлета) — 75 руб.
4. `salad` (салат) — 45 руб.
5. `tea` (чай) — 25 руб.
6. `compote` (компот) — 30 руб.
